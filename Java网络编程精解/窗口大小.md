窗口大小

```
当接收方发送一个ACK响应(即对收到数据的响应)时，接收方会告诉发送者下一次我能接收多少数据，我们管这个叫窗口大小(window size)。
一般这个窗口大小就是接收方缓冲区的大小。
```







````
只要接收方每次都正常响应一个ACK，这个发送窗口就会持续增长直到达到窗口大小的最大限制。
当接收方在一个往返时间内没有给发送方响应一个ACK，这个发送窗口大小将会被减少。
如果这个连接的传输过程中发生了网络拥塞，有可能数据包会被丢掉。为了解决这个，TCP设计了很多拥塞控制的算法，其中有一个叫做慢启动。
````



```
当接收方接收数据的速度大于它处理的速度时就会发生交通阻塞，缓冲区队列会达到极限，然后数据包将会被丢掉。
使用TCP慢启动，发送方的发送窗口大小最初会成指数增长（每次是之前的2倍），但是一旦丢了任何一个包，发送方的发送窗口大小将会减少到一个包。之后它将会再次指数级增长直到之前发生丢包时发送窗口大小的一半。在那时，发送窗口大小会由指数增长变为线性增长。
```
